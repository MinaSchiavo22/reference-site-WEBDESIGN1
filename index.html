Naming Conventions

Rules for naming files and paths

  • All Lowercase:Because different operating systems treat capitalization differently.
  • No spaces:Because URLs cannot have spaces in them.
  • Only letters, numbers, and dashes:Because URLs use a limited set of characters & dashes are word separatorswhile underscores are word connectors.

Commit messages

Commit messages have to be formal, and straight to the point. It is to write a short 3+ word description. Here are some examples for commits:

  • Fix: fixes bugs
  • Feat: introduces new features

These are used to create a code history.
